# pqaudit v1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a static Rust binary that scans live TLS endpoints, scores PQC readiness against NIST IR 8547 and CNSA 2.0, generates CBOM and SARIF output, and exposes an MCP server mode.

**Architecture:** A single Rust crate (lib + binary) organized into four pipeline stages: probe engine (raw TLS + rustls), audit engine (cert chain + findings + scoring), output formatters (JSON/SARIF/CBOM/human), and scanner orchestrator that wires them together with bounded tokio concurrency. MCP server is a Cargo feature that routes tool calls into the same scanner pipeline.

**Tech Stack:** Rust 1.85+, rustls 0.23 + aws-lc-rs (PQC handshake), tokio 1.50 (async), clap 4.6 (CLI), x509-parser 0.18 (certs), cyclonedx-bom 0.8 (CBOM), serde-sarif 0.8 (SARIF), owo-colors + indicatif (terminal), rmcp 1.2 (MCP, feature flag).

```rust
impl fmt::Display for TlsVersion {
    fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result {
        match self {
            Self::Tls13 => write!(f, "TLS 1.3"),
            Self::Tls12 => write!(f, "TLS 1.2"),
            Self::Tls11 => write!(f, "TLS 1.1"),
            Self::Tls10 => write!(f, "TLS 1.0"),

        }
    }
}
```

```rust
/// Shared trait for algorithm deadline tables. Lives here so all table
/// implementations (nist_ir8547, cnsa2, fips) import from a common location
/// rather than depending on each other.
pub trait DeadlineTable: Send + Sync {
    fn name(&self) -> &'static str;
    fn deadline_for(&self, alg: &AlgorithmId) -> Option<DeadlineInfo>;
    fn status_for(&self, alg: &AlgorithmId) -> AlgorithmStatus;
}

#[derive(Debug, Clone)]
pub struct DeadlineInfo {
    pub deprecated_year: u32,
    pub disallowed_year: u32,
    pub note: &'static str,
}

/// Compliance status of an algorithm under a given deadline table.
#[derive(Debug, Clone, PartialEq, Eq)]
pub enum AlgorithmStatus {
    Approved,
    Deprecated,
    Disallowed,
    Unknown,
```

```rust
// NOTE: NIST IR 8547 is an Initial Public Draft (November 2024).
// Deadline values below are from the IPD and may be revised before finalization.
use crate::AlgorithmId;
use crate::audit::tables::{DeadlineInfo, DeadlineTable};

pub struct NistIr8547Table;

impl DeadlineTable for NistIr8547Table {
    fn name(&self) -> &'static str { "NIST IR 8547 (IPD)" }

    fn deadline_for(&self, alg: &AlgorithmId) -> Option<DeadlineInfo> {
        Some(match alg {
            AlgorithmId::Rsa { min_bits } if *min_bits < 2048 => DeadlineInfo {
                deprecated_year: 2000, disallowed_year: 2000,
                note: "Already disallowed",
            },
            AlgorithmId::Rsa { min_bits } if *min_bits <= 3072 => DeadlineInfo {
                deprecated_year: 2024, disallowed_year: 2030,
                note: "Primary hard wall",
            },
            AlgorithmId::Rsa { .. } => DeadlineInfo {
                deprecated_year: 2024, disallowed_year: 2035,
                note: "Extended deadline for RSA-4096",
            },
            AlgorithmId::EcP256 | AlgorithmId::EcP384 => DeadlineInfo {
                deprecated_year: 2024, disallowed_year: 2030, note: "",
            },
            AlgorithmId::EcP521 => DeadlineInfo {
                deprecated_year: 2024, disallowed_year: 2035,
                note: "Extended deadline",
            },
            AlgorithmId::X25519 | AlgorithmId::Ed25519 => DeadlineInfo {
                deprecated_year: 2024, disallowed_year: 2030, note: "",
            },
            AlgorithmId::X448 | AlgorithmId::Ed448 => DeadlineInfo {
                deprecated_year: 2024, disallowed_year: 2035,
                note: "Extended deadline",
            },
            AlgorithmId::Dh { min_bits } if *min_bits < 2048 => DeadlineInfo {
                deprecated_year: 2000, disallowed_year: 2000,
                note: "Already disallowed",
            },
            AlgorithmId::Dh { min_bits } if *min_bits <= 2048 => DeadlineInfo {
                deprecated_year: 2024, disallowed_year: 2030, note: "",
            },
            AlgorithmId::Dh { .. } => DeadlineInfo {
                deprecated_year: 2024, disallowed_year: 2035,
                note: "Extended deadline for DH-3072+",
            },
            _ => return None,
        })
    }

    fn status_for(&self, alg: &AlgorithmId) -> AlgorithmStatus {
        // Derive status from deadline data relative to current year (2026 at time of writing)
        const CURRENT_YEAR: u32 = 2026;
        match self.deadline_for(alg) {
            None => AlgorithmStatus::Unknown,
            Some(d) if d.disallowed_year <= CURRENT_YEAR => AlgorithmStatus::Disallowed,
            Some(d) if d.deprecated_year <= CURRENT_YEAR => AlgorithmStatus::Deprecated,
            Some(_) => AlgorithmStatus::Approved,
        }
    }
}
```

Timeline multiplier (cert chain + cipher only):
```rust
pub fn timeline_multiplier(years_until_disallowance: i64) -> f32 {
    match years_until_disallowance {
        y if y >= 9  => 1.00,  // Safe
        y if y >= 5  => 0.75,  // Warning
        y if y >= 2  => 0.40,  // Urgent
        y if y >= 1  => 0.10,  // Critical (< 2 years, but not yet past)
        _            => 0.00,  // Non-compliant (y <= 0: deadline reached or passed)
    }
}
```

> Note: The spec's "< 2 years" zone means `y = 1` (one full year remains) → 0.10. The "Past deadline" zone means `y ≤ 0` (disallowance year ≤ current year) → 0.00. The match arm `y if y >= 1 => 0.10` correctly covers only `y = 1`, and the wildcard `_ => 0.00` correctly catches `y ≤ 0`. Years are integer-valued; there is no fractional case. When writing property tests, verify that `y = 0` returns 0.00 and `y = 1` returns 0.10.

```rust
use crate::ProbeResults;
use serde::{Deserialize, Serialize};

#[derive(Debug, Clone, PartialEq, Eq, Serialize, Deserialize)]
pub enum HndlRating { None, Low, Medium, High, Critical }

#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct HndlAssessment {
    pub rating: HndlRating,
    pub exposure_window_years: f32,
    pub cert_expires_before_q_day: bool,
    pub notes: Vec<String>,
}

/// Configuration for HNDL risk assessment.
#[derive(Debug, Clone)]
pub struct HndlConfig {
    pub q_day_year: u32,
    pub current_year: u32,
    /// Expiry year of the leaf certificate, if available. Per spec §8.4:
    /// cert expiry before Q-day may reduce the HNDL rating by one level
    /// because it forces a rekeying opportunity.
    pub cert_expiry_year: Option<u32>,
}

pub trait HndlModel: Send + Sync {
    fn name(&self) -> &'static str;
    fn assess(&self, probe: &ProbeResults, config: &HndlConfig) -> HndlAssessment;
}

pub struct DefaultHndlModel;

impl HndlModel for DefaultHndlModel {
    fn name(&self) -> &'static str { "default" }

    fn assess(&self, probe: &ProbeResults, config: &HndlConfig) -> HndlAssessment {
        // Extract group code from the PQC handshake result (or 0 if error)
        let group_code = probe.pqc_handshake.as_ref()
            .map(|r| r.negotiated_group.code_point)
            .unwrap_or(0);

        let exposure = (config.q_day_year as f32) - (config.current_year as f32);

        // cert expires before Q-day → forced rekeying opportunity → may reduce rating by one level
        let cert_before_q = config.cert_expiry_year
            .map_or(false, |y| y < config.q_day_year);

        let is_pure_pqc  = matches!(group_code, 0x0200 | 0x0201 | 0x0202);
        let is_hybrid_pqc = matches!(group_code, 0x11EB | 0x11EC | 0x11ED | 0x6399);

        let rating = if is_pure_pqc {
            HndlRating::None
        } else if is_hybrid_pqc {
            match exposure {
                e if e < 2.0 => HndlRating::Low,
                _             => HndlRating::Medium,
            }
        } else {
            match (exposure, cert_before_q) {
                (e, _) if e < 2.0        => HndlRating::Medium,
                (e, true) if e < 5.0     => HndlRating::Medium,
                (e, _) if e < 5.0        => HndlRating::High,
                _                        => HndlRating::Critical,
            }
        };

        HndlAssessment {
            rating,
            exposure_window_years: exposure,
            cert_expires_before_q_day: cert_before_q,
            notes: vec![],
        }
    }
}
```

```rust
/// Parse a TLS ServerHello or Alert from raw bytes.
pub fn parse_server_response(bytes: &[u8]) -> Result<ServerResponse, String> {
    if bytes.len() < 5 { return Err("too short".into()); }
    match bytes[0] {
        0x15 => Ok(ServerResponse::HandshakeFailure),  // Alert
        0x16 => parse_server_hello(bytes),             // Handshake
        _ => Err(format!("unexpected record type: 0x{:02x}", bytes[0])),
    }
}

/// Send ClientHello over an existing TcpStream, read response, classify it.
pub async fn probe_once(
    stream: &mut TcpStream,
    hello: &[u8],
    timeout_ms: u64,
) -> ServerResponse {
    use tokio::time::{timeout, Duration};
    if stream.write_all(hello).await.is_err() {
        return ServerResponse::ConnectionClose;
    }
    let mut buf = vec![0u8; 4096];
    match timeout(Duration::from_millis(timeout_ms), stream.read(&mut buf)).await {
        Ok(Ok(0)) => ServerResponse::ConnectionClose,
        Ok(Ok(n)) => parse_server_response(&buf[..n]).unwrap_or(ServerResponse::ConnectionClose),
        Ok(Err(_)) => ServerResponse::ConnectionClose,
        Err(_) => ServerResponse::Timeout,
    }
}
```

```rust
pub async fn pqc_probe(
    host: &str,
    port: u16,
    sni_override: Option<&str>,
    config: &ScanConfig,
) -> Result<PqcHandshakeResult, ProbeError> {
    let sni = sni_override.unwrap_or(host);

    // Build rustls config with aws-lc-rs crypto + PQC groups enabled
    let mut root_store = RootCertStore::empty();
    root_store.extend(webpki_roots::TLS_SERVER_ROOTS.iter().cloned());

    let tls_config = ClientConfig::builder_with_provider(
        rustls::crypto::aws_lc_rs::default_provider().into()
    )
    .with_root_certificates(root_store)
    .with_no_client_auth();

    let connector = TlsConnector::from(std::sync::Arc::new(tls_config));
    let stream = tokio::net::TcpStream::connect((host, port)).await
        .map_err(|e| if e.kind() == std::io::ErrorKind::ConnectionRefused {
            ProbeError::ConnectionRefused { host: host.into(), port }
        } else {
            ProbeError::TlsHandshakeFailed { reason: e.to_string() }
        })?;

    let server_name = rustls::pki_types::ServerName::try_from(sni.to_string())
        .map_err(|e| ProbeError::TlsHandshakeFailed { reason: e.to_string() })?;

    let tls_stream = connector.connect(server_name, stream).await
        .map_err(|e| ProbeError::TlsHandshakeFailed { reason: e.to_string() })?;

    let (_, session) = tls_stream.get_ref();

    let negotiated_suite = session.negotiated_cipher_suite()
        .ok_or_else(|| ProbeError::TlsHandshakeFailed { reason: "no cipher suite negotiated".into() })?;

    // Extract negotiated key exchange group from session
    // rustls exposes this via the peer_certificates and connection state
    // Group detection: parse the raw ServerHello key_share extension to get the group code point
    // (rustls doesn't expose this directly; read from HrrDetector state)

    let cert_chain = session.peer_certificates()
        .unwrap_or_default()
        .iter()
        .map(|c| c.as_ref().to_vec())
        .collect();

    // ... extract group code from connection state
    todo!("extract negotiated group code point from rustls session")
}
```

> **Note:** rustls 0.23 does not expose the negotiated NamedGroup directly via a public API. You will need to detect the negotiated group either by: (a) reading the key_share extension from the raw ServerHello before rustls processes it (requires a custom middleware), or (b) tracking which groups were offered in the ClientHello and which was returned in the ServerHello via HRR detection. Consult the rustls source and any available 0.23 `ConnectionCommon` introspection APIs. ## Quick scan

```bash
pqaudit example.com                         # JSON output, NIST scoring
pqaudit -o human example.com                # Colored terminal output
pqaudit --full-scan example.com             # Active cipher enumeration (~2min)
pqaudit --compliance cnsa2 example.com      # CNSA 2.0 scoring
pqaudit --fail-below 80 example.com         # CI/CD gate
pqaudit -o sarif example.com > scan.sarif   # For GitHub Advanced Security
pqaudit -o cbom example.com > cbom.json     # CycloneDX CBOM
```

## Score interpretation

| Score | Meaning |
|-------|---------|
| 80–100 | PQC-ready or nearly ready |
| 60–79 | Hybrid PQC negotiated, gaps remain |
| 40–59 | Classical TLS 1.3, no PQC |
| < 40  | Significant risk |

- [ ] **Step 3: Write .github/actions/pqaudit/action.yml**

```yaml
name: pqaudit TLS Scan
description: Scan TLS endpoints for post-quantum readiness
inputs:
  targets:
    description: Space-separated list of targets to scan
    required: true
  fail-below:
    description: Exit code 1 if score below this threshold
    default: "0"
  compliance:
    description: Scoring framework (nist, cnsa2, fips)
    default: nist
  output-format:
    description: Output format (json, sarif, human)
    default: sarif
runs:
  using: composite
  steps:
    - name: Download pqaudit
      shell: bash
      run: |
        curl -sSL https://github.com/YasogaN/pqaudit/releases/latest/download/pqaudit-linux-x86_64 -o pqaudit
        chmod +x pqaudit
    - name: Run scan
      shell: bash
      run: |
        ./pqaudit \
          --compliance ${{ inputs.compliance }} \
          --output ${{ inputs.output-format }} \
          --fail-below ${{ inputs.fail-below }} \
          ${{ inputs.targets }}
```
